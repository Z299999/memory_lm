# Orthfall Frontier World Bible

## 世界总览

这里是一个完全虚构的高维荒野生存世界：Orthfall Frontier，又称“折坠边疆”。

它不是普通荒野，也不是现实科幻设定的改写，而是一套刻意违背人类直觉的边境宇宙：

- 高地不一定比低地更冷，只有在某些朝向和光照状态下才成立
- 可见的水不一定能喝，不可见的冷凝反而常是唯一安全水源
- 某些营地在第一静止周期安全，但第二静止周期后会反转
- 一些生物追逐声音，另一些生物追逐沉默，另一些则追逐“重复的方向感”
- 路径不是固定空间，而是会在折层活性变化时改变“哪一条路径被算作同一条路”

实验结果统一为：

- `SAFE`
- `DANGEROUS`

这里的 `SAFE / DANGEROUS` 表示：

- 某次穿越、扎营、采集、储藏、发信号、取水、躲避或判断行为，是否会高概率导致迷失、坠入折层、遭遇生物、脱水、方向回路、时间错位或营地反噬。

Host 的任务是：

- 持续给出局部地貌、生物、资源和异常物理片段
- 迫使 tested agent 在高度超载的世界知识里提炼最关键的生存规则
- 避免把整个 frontier 的反直觉规律一次讲穿

## 基础规则

### 高频生存规则

1. 在 `玻风平滩`（`Glasswind Flats`）上，`splitlight` 通常比 `bright` 更危险，因为地表反射会制造错误方向感。
2. `虚苔细渠`（`Null Moss Rill`）附近看似潮湿，但其可见水常不适合直接采集；对 `霜凝水` 的误判会造成脱水风险。
3. `提灯漏谷`（`Lantern Sink`）会放大声音，因此任何 `signal` 行为在此区域通常更危险。
4. `白卷洞群`（`White Coil Hollow`）在第一静止周期内常被误判为理想营地，但久留通常风险更高。
5. `折幕帐`（`fold_tent`）只在 `stable` 或轻度 `warped` 折层中可靠；在 `shearing` 条件下风险显著上升。
6. `石倚棚`（`stone_lean_to`）通常比 `开敞篷布` 更安全，但在 `downpull` 风态下会积聚坠尘和生物。
7. `灰镜湖盆`（`Ash Mirror Lake`）在 `bright` 条件下会凝出可见水膜，但其边缘路径更容易错判深浅。
8. `静痕断带`（`Quiet Scar`）看上去安静，因此容易被当作扎营点，但其沉默区域会吸引追逐静默的生物。
9. `合唱蛾`（`choir_moths`）更容易被高频信号吸引，因此 `signal` 与 `辉藓` 同时出现时要谨慎。
10. `裂角鹿`（`rift_stags`）通常回避持续噪声，但会在 `still` 风态下接近无声营地。
11. `淡脂`（`pale_resin`）可用于短时密封 shelter，但在 `flare` 热态下会脆化。
12. `铰绳`（`hinge_rope`）适合跨越普通裂隙，但在 `第九折裂谷` 的双路径区不可靠。

### 方向与折层规则

13. `蓝脊台缘`（`Blue Spine Shelf`）在 `bright` 条件下虽然视野好，但气温和高度感常反转，容易误判补给节奏。
14. `镜盐`（`mirror_salt`）能帮助稳定部分方向标记，但在 `crosswind` 下容易形成误导性晶线。
15. `漂移雪橇`（`drift_sled`）适合平直区域，不适合进入 `墨铰山梁` 或 `淡梯沙丘`。
16. `埋藏补给点`（`buried_cache`）通常是安全的补给策略，但在 `记雾` 活跃区域容易被自己遗忘。
17. `漂礼残殿`（`Drift Chapel Ruin`）会发出稳定视觉参照，因此常被当作安全地标，但在 `warped` 折层中会延迟空间反馈。
18. `返步线`（`returning_line`）出现时，`uncertain` 很快会转为 `looping`，此时继续 `cross` 通常危险。
19. `折帘湿原`（`Foldveil Marsh`）会改变步幅感，在没有可靠锚点时长距离穿越通常危险。
20. `逆昼台`（`Counterday Ledge`）的阴影方向可能与主光源相反，因此不能用常识判断取水或避热位置。

### 生物与资源规则

21. `树脂封缝`（`resin_seal`）对 `帷孢` 有一定阻隔作用，但不能抵御 `铰虫` 的结构破坏。
22. `灰花异林`（`Cinder Bloom Grove`）在 `temperate` 时相对可穿越，在 `flare` 时会释放高风险孢尘。
23. `坠雨`（`downfall_rain`）不一定意味着补水机会；某些区域的降雨会加重折层活性。
24. `静幕布`（`still_fabric`）在 `crosswind` 下非常有用，但在 `still` 条件下会积聚静电标记。
25. `哑鳐`（`mute_ray`）会避开持续低频振动，所以在 `震律场` 扎营有时反而更安全。
26. `热髓`（`heat_marrow`）能在 `cold` 条件下保命，但会提高某些夜行生物的感知概率。
27. `阴凝珠`（`shade_condensate`）常出现在反直觉阴影面，是少数稳定补水来源，但只在特定光照下成立。
28. `脉纹钉`（`vein_spike`）能固定方向参照，但会招来感知微震的生物。

### 复合判定规则

29. 若一个方案同时命中“方向错判风险 + shelter 反转风险 + 生物吸引风险”，默认判为 `DANGEROUS`。
30. 若一个方案命中“折层不稳 + 第二静止周期 + 低可靠度方向”，默认判为 `DANGEROUS`。
31. 若某路径需要在 `looping` 状态下继续 `cross` 才能完成，通常应直接判为 `DANGEROUS`。
32. 若资源采集行为同时触发“强光诱导 + 反射诱捕 + 边缘深浅错判”，默认判为 `DANGEROUS`。

## 关键例外

1. 在 `静痕断带`（`Quiet Scar`），`开敞篷布 + still + dim` 并不安全；这里的沉默会吸引 `裂角鹿`，通常判 `DANGEROUS`。
2. `提灯漏谷`（`Lantern Sink`）中的 `signal` 几乎总是 `DANGEROUS`，即便你使用的是低光信号而不是高声信号。
3. 在 `灰镜湖盆`（`Ash Mirror Lake`），`bright` 条件下若只进行 `collect` 而不尝试 `cross`，通常可以视为 `SAFE`。
4. `白卷洞群`（`White Coil Hollow`）在第一静止周期内使用 `石倚棚` 短暂停留通常 `SAFE`，但第二静止周期后会转为 `DANGEROUS`。
5. `震律场 + 静幕布 + 低噪声` 通常不安全，但若同时存在持续低频震动，可暂时驱离 `哑鳐`，形成短时 `SAFE` 窗口。
6. `蓝脊台缘` 在 `bright + cold` 下并不总危险；若携带 `热髓` 且只进行短距离 `cross`，通常 `SAFE`。
7. `镜盐` 在 `淡梯沙丘` 中大多误导，但若风态为 `still` 且光照为 `dim`，可形成少数可靠标记。
8. `埋藏补给点` 在 `记雾` 活跃区通常没用，但若藏于 `脉辉边坡` 并配合 `辉藓` 标记，反而较安全。
9. `灰花异林` 中 `树脂封缝` 一般不足以隔绝孢子；但若热态为 `cold` 且只短时通过，则通常 `SAFE`。
10. `第九折裂谷` 使用 `铰绳` 几乎总是危险，除非折层活性正好是 `stable` 且只跨越外缘第一折。
11. `漂礼残殿` 在 `warped` 时危险，但在 `stable + dim` 条件下是少数可信的夜间定位点。
12. `虚苔细渠` 的可见水通常不安全，但清晨冷凝出的 `霜凝水` 若来自石缘结珠，可安全收集。
13. `玻鬣豺`（`glass_jackals`）会追逐反射而非肉味，因此在 `玻风平滩 + bright` 下携带镜面器物几乎总是 `DANGEROUS`。
14. `折幕帐` 在 `shearing` 折层下通常危险，但若放置于 `蓝脊台缘` 外缘偶发锚区，才可能稳定。
15. `downpull` 风态并不总坏；在 `朱齿岩脊`，它会压低热浪，使短时 `collect` 反而更安全。
16. `逆昼台` 的阴影常用于判断错误方向，但在 `splitlight + cold` 下，阴影面形成 `阴凝珠` 时反而是安全补水点。
17. `折帘湿原` 看似富水，但若 `direction_reliability=looping`，停下来 `camp` 反而通常更安全，继续 `cross` 更危险。
18. `骨砂港` 的回响通常吸引捕食者，但在 `crosswind + bright` 中，噪声被吹散，短时 `collect` 可视为少数 `SAFE` 窗口。
19. `眠脉峡` 在 `first_rest` 看似稳定，但若带有 `热髓` 并进入 `second_rest`，会触发延迟热流，通常 `DANGEROUS`。
20. `卷石压棚`（`coil_stone_shelter`）在 `downpull + warped` 条件下通常优于 `折幕帐`，是少见的正例外。

## Host 绝不能直接泄露的信息

- 不要一次把所有“看起来安全、其实危险”的反直觉地点都列出来。
- 不要直接告诉 tested agent 哪些区域会在第二静止周期后反转。
- 不要把所有生物“追逐什么”的模式一次说全。
- 不要把折层机制和方向可靠度的全局规律完整讲穿。
- 不要直接给出完整的“万能求生法则”，因为这个世界刻意存在大量局部例外。

## 可供抽题的任务素材

推荐题型：

- “某人是否应在此时此地扎营？”
- “某个区域是否适合发信号？”
- “在给定资源和条件下，这次收集 / 穿越 / 埋藏补给是否 `SAFE`？”
- “上一轮只改一个关键条件，例如从 `bright` 改成 `splitlight`，或从 `first_rest` 改成 `second_rest`，看 agent 是否注意到变量。”

推荐出题策略：

1. 前期先让 agent 建立基本生存直觉。
2. 中期开始用“看起来更安全其实更危险”的地貌和营地反转打破现实直觉。
3. 后期反复测试那些只在单一条件下才安全的关键例外。
4. 不要总围绕同一个区域，应迫使 memory 压缩为跨区域规则。

示例任务种子：

- 在 `提灯漏谷` 中用 `signal` 请求援助
- 在 `灰镜湖盆` 的 `bright` 条件下只收集表层水
- 在 `白卷洞群` 中第二静止周期继续 `camp`
- 在 `静痕断带 + still + dim` 下用 `开敞篷布` 扎营
- 在 `蓝脊台缘 + bright + cold` 下携带 `热髓` 短程穿越
- 在 `淡梯沙丘` 中使用 `镜盐` 标记路径
- 在 `逆昼台 + splitlight` 下尝试从阴影面收集 `阴凝珠`
- 在 `骨砂港 + crosswind` 中短时采集 `卷石`

## 判分原则

- 第一版统一只接受 `SAFE` 或 `DANGEROUS`。
- 若存在关键例外，以关键例外优先。
- 不是“传统荒野常识对不对”，而是“是否符合 Orthfall 的反直觉物理、生物和折层规则”。
- 若 tested agent 把现实经验机械套进来，通常应被判错。
- 只改一个环境变量导致结论翻转，是这个 world 最重要的考点之一。

## Host 行为约束

- 保持地貌、生物、资源和折层规则一致，不要临时发明新的区域物理。
- 可以逐轮暴露局部反直觉性，但不要一下把整个 frontier 的隐藏规律讲完。
- 尽量让相邻轮次共享区域或资源，只改一个关键条件，迫使 agent 学规则而非背答案。
- 反馈可指出 agent 忽略了“风态、折层、静止周期、方向可靠度、生物偏好”，但不要总给完整规则。
