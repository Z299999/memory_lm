# Court of Veils World Bible

## 世界总览

这里是一个完全虚构的叙事政治世界：The Court of Veils，又称“纱幕王廷”。

它不是现实中任何朝代、宫廷或贵族系统的映射，而是一套被刻意设计得高度反直觉的政治-礼制宇宙。在这个世界里：

- 身份高不一定意味着安全，反而可能意味着更高的可追踪性
- 私下会面不一定比公开场合安全，有些人物没有见证反而更危险
- 誓约不是抽象承诺，而是会反噬、绑定、串联旧案的真实结构
- 谣言不是背景噪音，而是会改变风险本身的“热态场”
- 同一段话、同一件礼物、同一种接触方式，在不同节日、时段、见证密度下可能彻底翻转

实验结果统一只允许二分类：

- `SAFE`
- `DANGEROUS`

这里的 `SAFE / DANGEROUS` 表示：

- 某次会面、传话、赠礼、誓约、告发、托付或公开举动，是否会高概率触发身份暴露、礼制追责、誓约反噬、旧案联动、被监听、被栽赃或阵营崩塌。

Host 的任务不是一次性把全部人物关系和规则讲明，而是：

- 逐轮暴露人物、旧案、家族关系、传闻和礼制片段
- 让 tested agent 在信息严重过载的前提下，逐渐学会保留高价值条件与关键例外
- 用“几乎相同、但只改一个礼制条件或见证条件”的题面测试 memory 的压缩质量

### 社交载体

这些不是物理容器，而是信息和关系交换的形式：

- `sealed_note`：封缄便笺
- `spoken_whisper`：贴近耳畔的低语
- `mirror_oath`：镜誓
- `public_toast`：公开祝辞
- `gift_token`：礼符或礼物信物
- `stage_lyric`：戏词或歌句
- `witness_seal`：见证印记
- `ledger_mark`：账册批记

### 情境条件

- 见证密度：`private` / `watched` / `ceremonial`
- 谣言热度：`cold` / `active` / `fevered`
- 时段：`dawn` / `dusk` / `midnight`
- 礼制状态：`ordinary` / `festival` / `mourning`
- 档案敏感度：`low` / `flagged` / `sealed`
- 旧案邻近度：`distant` / `nearby` / `direct`

### 操作

- `confide`
- `deliver`
- `swear`
- `toast`
- `accuse`
- `meet`
- `witness`
- `redirect`

## 基础规则

### 高频礼制规则

1. 任何通过 `spoken_whisper` 传递的敏感信息，在 `watched` 条件下通常更危险。
2. `public_toast` 在 `festival` 状态下会被更多人转述，因此任何涉及秘密联盟的表达都风险更高。
3. `sealed_note` 在 `private` 情境下一般比 `spoken_whisper` 更安全，但若接收者隶属 `门账署` 或 `墨屋`，风险会上升。
4. `mirror_oath` 会强化承诺可信度，但一旦对象间存在旧债或旧誓冲突，反噬风险显著提高。
5. `戏环` 成员通过 `stage_lyric` 交流时，在 `festival` 中通常更安全，因为信息更容易被误判为表演。
6. `灯廷` 的人不适合在 `midnight` 以 `spoken_whisper` 形式与 `灰旗` 旧部接触，这会引发监听怀疑。
7. `mourning` 期间的公开社交更受注目，任何越礼行为都更危险。
8. 若一个人同时与 `墨屋` 和 `戏环` 都有未公开联系，则其低调会面风险默认上升。
9. `gift_token` 一般比直接告密安全，但若礼物带有明确阵营标记，就会成为追踪线索。
10. `镜档馆` 出身人物在 `mirror_oath` 场景里通常更可信，但也更容易被旧档案反咬。
11. `盐区` 成员在 `dusk` 发起交易相关接触通常较安全，在 `midnight` 则更容易被解读为密谋。
12. `纱修会` 见习者在誓约传播中往往不是安全中介，因为他们容易在礼制上犯错。

### 结构性风险规则

13. 若谣言热度为 `fevered`，任何与旧王、放逐名单、王弟旧府相关的会面都更危险。
14. `门账署` 成员适合公开、程序性的接触，不适合私下中转敏感消息。
15. `戏环` 与 `墨屋` 的跨阵营接触，若通过 `public_toast` 进行，通常极不安全。
16. `ordinary` 状态下的 `sealed_note` 是最常见的低风险基线，但不能覆盖关键例外。
17. 与 `阿里克旧府` 有关的人物在旧案相关话题中会带额外风险，因为他们容易触发旧忠诚审查。
18. `钟塔` 与 `灯廷` 之间的联系通常意味着时序与巡逻记录会被核对。
19. `gift_token` 若通过 `萨瓦·诺尔` 这类平民网络中介流转，通常比通过官署人物更隐蔽。
20. `festival` 会掩盖一部分公开噪声，但不会掩盖誓约类违规。
21. `mourning` 状态下的 `stage_lyric` 比平时更危险，因为它会被视为不敬或暗号。
22. `private` 不等于安全；对某些人物而言，没有见证反而更危险，因为事后无法自证。
23. `witness_seal` 通常提高形式安全性，但如果见证人属于错误阵营，会导致额外风险。
24. `ledger_mark` 对普通商业事务有保护作用，但一旦话题涉及旧案，它会留下更难抹去的轨迹。

### 复合判定规则

25. 若一个场景同时命中“誓约风险 + 谣言高热 + 礼制异常”三类规则，默认判为 `DANGEROUS`。
26. 若一个行动同时触发“旧案联想 + 阵营错位 + 礼制不合”三重怀疑，默认判为 `DANGEROUS`。
27. 若 `档案敏感度=sealed` 且参与者之一来自 `镜档馆`，则任何私下重定向信息的行为都至少提升一档风险。
28. 若 `旧案邻近度=direct`，则即使是形式上正规的 `witness` 也未必安全。
29. 若同一人物在短时间内同时使用两种社交载体传递相近内容，风险会上升，因为轨迹容易交叉验证。
30. 若一个行为既能被解释为礼貌，也能被解释为政治暗示，则在 `fevered rumor` 下默认按危险解释处理。

## 关键例外

1. `维墨琳·沃斯`（`Maelin Voss`）与 `利欧·梅罗`（`Lio Merrow`）之间通过 `sealed_note` 传递档案信息，在 `ordinary + private + archive_sensitivity=low` 下通常 `SAFE`，即使谣言热度为 `active`。
2. `伊瓦拉·昆`（`Ivara Quen`）通过 `public_toast` 向 `盐区` 自家网络发出商业暗示，在 `festival` 中通常 `SAFE`，因为这符合她的公开身份。
3. `维拉·索尔`（`Veyra Sol`）与 `瑟莎·乌姆`（`Thessa Ulm`）之间的 `stage_lyric` 在 `festival + watched` 下反而更安全，因为旁观者会把异常理解成戏码的一部分。
4. `泰伦·灰坠`（`Teren Ashfall`）只要在 `midnight` 通过 `spoken_whisper` 接触 `灯廷` 人员，就几乎总是 `DANGEROUS`，无论见证密度如何。
5. `米瑞丝·索恩`（`Mireth Sorn`）参与的 `mirror_oath` 若对象之一是 `德恩·阿里克`（`Deln Aric`），默认 `DANGEROUS`，因为会触发旧王誓系回响。
6. `哈罗·凯斯特`（`Harrow Kest`）作为信使时，用 `sealed_note` 送普通礼单通常安全，但若收件人是 `欧伦·佩尔` 且谣言热度为 `active`，则变为 `DANGEROUS`。
7. `伊瑟尔·摩尔`（`Yselle Mor`）因为 oath-neutral，其参与 `gift_token` 转交时通常较安全，除非礼制状态为 `mourning`。
8. `鲁凡·艾德`（`Ruvan Elde`）与任何人进行 `private` 会面，只要话题涉及旧账册和放逐名单，默认 `DANGEROUS`。
9. `菲拉·黛恩`（`Fira Dane`）通过 `sealed_note` 给 `盐区` 传送路线修改，在 `dawn + ordinary` 下通常 `SAFE`，这是少数稳定的边缘通信例外。
10. `埃尔里克·芬`（`Elric Fen`）在 `ceremonial` 环境中发起 `accuse` 并不一定危险；若对象是 `乔林·派克` 且有合法文印支撑，通常 `SAFE`。
11. `萨瓦·诺尔`（`Sava Noll`）代转 `gift_token` 给 `维拉·索尔` 在 `festival` 中通常安全，但若谣言热度升到 `fevered`，会转为 `DANGEROUS`。
12. `凯尔·钟恩`（`Cael Durn`）与 `瑟里特·维尔`（`Serit Vale`）的 `midnight meet` 通常安全，因为守钟与掌灯在夜间存在正当重叠职责。
13. `阿里克旧府` 相关人物在 `mourning` 期间接受 `public_toast` 几乎总是 `DANGEROUS`，哪怕内容本身无害。
14. `奈瑞丝·荆恩`（`Neris Thane`）若以医师身份在 `private` 条件下与 `米瑞丝·索恩` 会面讨论伤病，通常 `SAFE`，即便修会本身敏感。
15. `维墨琳·沃斯` 通过 `mirror_oath` 与 `埃尔里克·芬` 交换信息并不安全，哪怕两人都来自文书系统，因为法印和档案誓约会互相锁定。
16. `露安·凯维`（`Luan Kevi`）在 `festival` 中向 `瑟里特·维尔` 发出 `public_toast` 通常安全，但若同时 `old_case_proximity=nearby`，则会被解读为派系试探。
17. `赫缇·桑若`（`Heti Sanro`）若以 `ledger_mark` 形式记录放逐名单相关事项，在 `private` 条件下并不更安全，反而通常更危险。
18. `尤娜·佩瑟`（`Yuna Paether`）主持的 `witness_seal` 在 `mourning` 中通常不安全；但若用于纯医疗通行证明，可视为少数 `SAFE` 例外。
19. `卡德·维森`（`Kade Vessen`）通过 `gift_token` 与 `盐区` 自身网络往来通常安全，但若对象同时与 `墨屋` 有联系，则立即升高风险。
20. `塔尔·欧里斯`（`Tal Oris`）在地窖中转述秘密通常危险，但若只是转移“众所周知但未正式宣布”的酒宴座次调整，则通常 `SAFE`。

## Host 绝不能直接泄露的信息

- 不要一次性列出人物之间的全部隐秘关系。
- 不要直接告诉 tested agent 哪些人物“本质上互信”。
- 不要把关键例外整理成显式清单给出。
- 不要把 `阿里克旧府`、旧王、放逐名单之间的深层联系完整说穿。
- 不要把“哪些公共行为只是伪装安全”总结成全集。
- 不要把 `档案敏感度` 与 `旧案邻近度` 的全局作用一次讲透。

## 可供抽题的任务素材

每轮任务仍推荐保持二分类，适合的题型包括：

- “在某个礼制状态和谣言热度下，某人是否适合以某种方式接触另一人？”
- “某个礼物、誓约、密语、公开致辞或见证安排是 `SAFE` 还是 `DANGEROUS`？”
- “上一轮 agent 忽略了见证密度、档案敏感度或旧案邻近度，本轮改一个条件重新考。”

推荐出题策略：

1. 前期先从高频礼制规则入手。
2. 中期逐渐引入旧案、旧誓、阵营错位和档案敏感度。
3. 反复比较 `private` 与 `watched`，避免 agent 把“私下”机械等同于“安全”。
4. 后期多出角色高度相似、但因旧案位置不同而结论翻转的题。
5. 不要连续多轮只考同一人物，应迫使 agent 学会跨人物、跨礼制抽象。

示例任务种子：

- `泰伦·灰坠` 在 `midnight + watched` 中对 `瑟里特·维尔` 发起 `spoken_whisper`
- `维拉·索尔` 在 `festival + watched` 中向 `瑟莎·乌姆` 传递 `stage_lyric`
- `维墨琳·沃斯` 在 `ordinary + private` 下向 `利欧·梅罗` 送 `sealed_note`
- `米瑞丝·索恩` 与 `德恩·阿里克` 进行 `mirror_oath`
- `萨瓦·诺尔` 在 `festival` 中代转 `gift_token` 给 `维拉·索尔`
- `鲁凡·艾德` 在 `private + active rumor` 下约见 `伊瓦拉·昆`
- `赫缇·桑若` 在 `sealed archive + private` 条件下进行 `ledger_mark`
- `尤娜·佩瑟` 在 `mourning + ceremonial` 条件下发起 `witness_seal`

## 判分原则

- 第一版统一只接受 `SAFE` 或 `DANGEROUS`。
- 核心不是“道德上好不好”，而是“是否会导致高概率的暴露、追踪、誓约反噬或政治性后果”。
- 若同时命中基础规则和关键例外，以关键例外优先。
- 若 tested agent 把 `private` 机械等同于 `SAFE`，通常应判错。
- 若任务明显涉及高礼制敏感期、旧案联动和档案敏感度叠加，应提高危险判定权重。

## Host 行为约束

- 保持人物关系、旧案、礼制与誓约逻辑一致，不要临时发明新血缘或新誓系。
- 可以逐轮暴露“谁看起来像盟友但其实不能私下接触”的反直觉结构。
- 反馈可指出 agent 忽略了“礼制、见证、旧案、档案敏感度、阵营错位”，但不要总把完整真相讲透。
- 尽量让 tested agent 从零散场景中压缩出稳定的社交风险规则，而不是只记住某个人名。
