# exp0415 — MLP Online Learning Smoke Test

`exp0415_memoryforMLP` 验证一个核心问题：**MLP 能否通过在线学习（online learning）在权重里编码序列记忆？**

实验设置极简：

- 输入永远是常量 `1`（代表 `<BOS>`）
- 目标序列是 `0, 1, 0, 1, 0, 1, ...`（无限交替）
- MLP 架构：`[1, 4, 4, 1]`（1 输入，两层 hidden 各 4 个节点，1 输出）
- 每个时刻做一次前向传播 + 一次反向传播 + 权重更新（batchsize=1）
- MLP 没有任何显式记忆单元，唯一的"记忆"是权重本身

**目的**：作为对照基线，为后续引入 simplex/triangle 网络做铺垫。

## 实验结论

**MLP 无法通过 SGD 在线学习可靠地编码交替序列记忆。**

### 不同学习率下的行为

| LR | 现象 | 原因 |
|---|---|---|
| 低（0.1） | 输出收敛到 `0.5` | 相邻步骤的梯度方向相反，来回抵消，网络取两个目标的平均值 |
| 中（0.5–2） | 快速塌缩到 `0` 或 `0.5` | 权重更新后在饱和区附近振荡，幅度不断衰减 |
| 高（≥5） | 输出塌缩到 `0`，loss 在 0/1 之间振荡 | 过大的权重更新把网络压入 sigmoid 饱和区，梯度无法逃逸 |

### 根本原因（解析验证）

对单层网络 `output = σ(w + b)` 做解析分析：

- 时刻 t，目标为 `0`，梯度让 `w` 减小 → 下一步 t+1，目标为 `1`，但输出已经更接近 `0`，反而更难达到目标
- 梯度更新**永远服务于当前步**，但这恰好**伤害了下一步的预测**
- 具体地：用不同 LR 验证后，没有任何 LR 能让训练 target=0 之后自然地让下一步输出接近 `1`

```
lr=  0.5  y0=0.622  y1_pred=0.552  (need y1≈1)  ok=YES  # 微弱改善，远不够
lr=  1.0  y0=0.622  y1_pred=0.479  (need y1≈1)  ok=no
lr=  5.0  y0=0.622  y1_pred=0.081  (need y1≈1)  ok=no
lr= 10.0  y0=0.622  y1_pred=0.005  (need y1≈1)  ok=no
lr= 50.0  y0=0.622  y1_pred=0.000  (need y1≈1)  ok=no
```

### 解读

MLP 的前向输出只依赖于当前权重，与时刻 t 无关。在 constant input 下，相邻时刻的输出差异**只能来自权重更新**。而标准 SGD 的权重更新方向永远是"减小当前误差"，这个方向和"增大下一步的输出振幅"是冲突的。网络最终收敛到某个稳定的固定点（`0` 或 `0.5`），而不是振荡点。
