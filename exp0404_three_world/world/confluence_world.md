# Confluence World Bible

## 世界总览

这里是一个统一的大世界：Confluence，又称“汇流界”。

它不是第四个独立 world，而是此前三个切面的完整版：

- [`court_of_veils_world.md`](./court_of_veils_world.md) 负责王廷、人物关系、礼制与誓约
- [`orthfall_frontier_world.md`](./orthfall_frontier_world.md) 负责边疆、路线、生存与反直觉自然法则
- [`asterion_lab.md`](./asterion_lab.md) 负责试剂、容器、材料反应与延迟翻转

在汇流界里，这三套规律从来不是分开的。

- 王廷会派人运送试剂、样本、誓约载体或伪装成礼物的敏感物
- 这些东西要穿过折坠边疆，途中会受地貌、补给、静止周期和异常压力影响
- 载荷到达后，是否还能按礼制交接、能否安全会面、会不会暴露身份或触发反应，取决于人物关系与载荷状态的共同作用
- 路上临时采集的资源、偶遇的见证人、被迫避险的营地、改用的容器，都可能让原本安全的任务在最后一刻翻转

实验结果统一只允许：

- `SAFE`
- `DANGEROUS`

这里的 `SAFE / DANGEROUS` 指的是：

- 当前决策是否会高概率导致交接失败、人物暴露、礼制追责、材料失稳、环境致死、旧案联动、誓约反噬或后续链条不可控。

Host 的任务不是一次把全部规律讲透，而是：

- 每轮只暴露一段任务链中的局部条件
- 让 tested agent 在严重信息超载下逐渐压缩出高价值规则
- 反复用“几乎一样，但少改一个人 / 一种材料 / 一段路线 / 一个礼制时点”的题面测试 memory 是否真的在进化

## 世界统一设定

汇流界有三层彼此纠缠的秩序：

### 1. 王廷秩序

王廷通过礼制、誓约、档案与见证维持表面秩序。

- 谁能见谁
- 什么东西可以公开携带
- 哪种会面必须见证
- 哪类载荷会被当作政治暗示
- 哪些人物一旦在错误时点出现，就会触发旧案联动

### 2. 边疆秩序

折坠边疆不服从常识。

- 路线会变
- 地貌会翻转
- 可见的水不一定安全
- 安全营地可能只在第一静止周期安全
- 某些声音、反射、静默或方向感本身就会招来危险

### 3. 载荷秩序

秘研材料不是被动物件。

- 它们会被光、温、湿、压、气流、停留时间、容器与路径共同改变
- 某些载荷本身稳定，但不适合被某些人携带
- 某些载荷在实验室安全，在边疆中会翻转
- 某些载荷即使物理上稳定，也会让会面失去礼制合法性

在汇流界里，一次任务必须同时通过这三层秩序，才算真正 `SAFE`。

## Tested Agent Role

tested agent 是“边廷转运审查官”。

这个角色设定应长期稳定：

- 身份：负责审查从王廷、秘研设施、边疆前哨和中转站之间流动的人、物、路线与交接窗口的低阶官员。
- 权限边界：只能看到本轮下发的局部转运指令、局部人物背景、局部路线条件、局部载荷说明，以及自己过去留下的外部 memory。
- 不知道的内容：完整人物暗线、完整材料手册、完整折层地图、所有旧案、全部安全窗口。
- 工作目标：判断某个阶段性的任务决策是否应放行，例如是否继续运送、是否中途采集、是否更换容器、是否照原计划会面、是否该立刻撤回。
- 说话风格：简洁、实用、像一个天天给高风险任务盖章或拦截的基层审查官。
- memory 风格：应越来越像“转运放行笔记”，记录的重点应是人物禁配、载荷翻转点、路线窗口、礼制陷阱与跨域例外。

Host 在构造 `AGENT_INPUT` 时，应默认让 tested agent 以这个身份接任务，例如：

- 某人是否应携带某物走某条路
- 某物在途中被迫改容器后，是否还应继续送达
- 某次会面在当前礼制状态、地貌状态和载荷状态下是否应立刻取消

## 任务链结构

汇流界的基本单位不是静态场景，而是一条任务链。

一条典型任务链包含：

1. `委托方`
   - 谁下令，任务表面名义是什么
2. `携带者`
   - 谁在送、谁在陪同、谁在监督
3. `核心载荷`
   - 试剂、样本、礼物、文书、誓约载体、混合载荷
4. `容器与封装`
   - 原始容器是什么，是否中途替换，是否需要隐藏
5. `路线`
   - 要经过哪些边疆区域，当前环境如何
6. `途中事件`
   - 遇人、遇险、采集、扎营、改道、泄密、检查点、设备损坏
7. `目标接触`
   - 要见谁，在哪里见，以什么形式见
8. `当前判定节点`
   - 问题不是整条任务是否完美，而是“现在这一步是否还应放行”

## 关键节点

### 王廷端

主要负责：

- 发起任务
- 给任务套上礼制名义
- 决定谁可以公开出现，谁只能暗中出现
- 决定一份载荷是“礼物”“医用物”“档案物”还是“不可见的东西”

高频人物仍主要来自 Court of Veils，例如：

- `维墨琳·沃斯`（`Maelin Voss`）
- `瑟里特·维尔`（`Serit Vale`）

- `奈瑞丝·荆恩`（`Neris Thane`）
- `埃尔里克·芬`（`Elric Fen`）

### 中转与边疆端

主要负责：

- 运送
- 护送
- 改道
- 补给
- 扎营
- 临时取水与采集
- 决定是“保任务”还是“保命”

高频地貌与边疆条件主要来自 Orthfall Frontier，例如：

- 容器要求
- 反应窗口
- 延迟风险

## 核心跨域规则

### 总体规则

1. 一条任务链只要在“人物合法性、路线生存性、载荷稳定性”三者中的任一处高概率失败，就应判为 `DANGEROUS`。
2. 在汇流界里，路线不是背景，人物不是背景，容器也不是背景；三者都属于判题核心条件。
3. “能送到”不等于“能交付”，很多任务在最后一步才因为会面语境或礼制失效而翻转。
4. “人活着”不等于“任务安全”，很多保命动作会污染载荷或破坏交接合法性。
5. “材料稳定”不等于“会面安全”，某些材料会让错误人物携带它这件事本身变危险。

### 人物与载荷规则

6. 来自 `镜档馆`、`印署`、`灯廷` 的人物携带高可见度载荷时，通常比边疆人物更容易留下可验证轨迹。
7. 与旧王旧案、放逐名单或誓约残脉有关的人物，不应携带会在 `delayed` 阶段翻转的载荷进入正式接触。
8. 某些人物适合传话，不适合传物；尤其当载荷需要稳定容器或会在途中产生光、热、气味、回响时。
9. 高身份人物亲自携带敏感载荷通常比派低关注人物中转更危险。
10. 一次任务若同时包含政治意图和技术载荷，默认会被更严格地解读。
11. 同一人物在短时间内既做关系接触又做载荷交接，风险会显著上升。
12. 若目标人物本身对污染、旧案、誓约或形式瑕疵高度敏感，则中途发生过的临时处理会放大最终风险。

### 路线与载荷规则

13. 任何受光、温、湿、压、气流或静置阶段影响的载荷，都必须把路线视为反应条件的一部分。
14. 某些在 Asterion 内安全的组合，一旦经过 `splitlight`、`warped`、`pulse frostfall`、`under-silt drift` 或 `loud sky-noise` 环境，就不再能按实验室直觉判断。
15. 若载荷依赖原始容器稳定，而路线包含强制扎营、紧急撤离、坠落、浸湿或补给紧张，则风险显著上升。
16. 若补给状态降至 `strained` 或 `critical`，任何需要精确温压控制的载荷都更容易翻转。
17. 路上采集的冷凝物、阴凝珠、镜盐、辉藓、冷箔片等物，不应被视作单纯补给；它们可能改变主载荷状态。
18. 若任务必须跨过 `first_rest` 进入 `second_rest`，而载荷存在 `delayed` 风险，则“今晚还能撑住”不等于“交付仍然安全”。
19. 透明容器、银封容器、琥珀容器在边疆中的表现会受到光照、湿态和颠簸共同影响，不能只按实验室规则判断。
20. 某些边疆资源能短时稳定载荷，但同时会抹掉“原样交付”的形式合法性。

### 人物与路线规则

21. 对某些任务来说，最危险的不是地貌本身，而是路上被谁看见、谁能把这次出行和谁联系起来。
22. 经过会放大声音、反射、静默痕迹或方向失真的区域时，必须同时考虑生存风险与身份暴露风险。
23. 在 `festival`、`mourning`、`ceremonial` 等时段里，路线延误会改变最终会面的意义。
24. `checkpoint` 和 `unexpected_witness` 经常比地貌本身更能毁掉一次原本技术上安全的交付。
25. 遇到与目标人物同阵营的人通常不代表更安全；阵营内部的交叉验证本身就是风险。
26. 若任务表面名义是 `medical` 或 `supply`，改道有时可被接受；若名义是 `ritual` 或 `archive`，改道往往会破坏合法性。

### 途中采集与会面翻转规则

27. 为保命而进行的采集、取水、短时封装或临时降温，可能让人活下来，却让目标人物拒绝接收载荷。
28. 某些临时采集物对环境风险有帮助，却会改变主载荷气味、湿态、光学表现或容器纯度。
29. 若一次任务同时发生“路线翻转、载荷翻转、会面语境翻转”三种变化，默认判为 `DANGEROUS`。
30. 路上顺手采集到的额外物，若和最终交接对象所属系统不兼容，通常会把安全会面翻成危险。
31. 若一段旅程中既发生了材料状态变化，又发生了人物关系变化，不应只用单域规则解释，默认风险上升。
32. 若任务的直接目标从“保密交付”被迫改成“公开通行”，有时能保命，但这类成功不等于原任务仍然安全。

### 形式与合法性规则

33. 对王廷与档案体系而言，容器、封装、见证方式本身就是内容的一部分。
34. 临时换容器、临时稀释、临时封缝，哪怕能稳定载荷，也可能使载荷不再被视为“同一件东西”。
35. 某些载荷可以秘密送达，却不能被公开接收；另一些载荷可以公开接收，却不能秘密运送。
36. 若一次交接既没有物理稳定性，也没有礼制合法性，则应直接判为 `DANGEROUS`，无需再比较哪种失败更严重。

### 跨域改写规则

37. 若任务名义从 `medical` 或 `supply` 在途中被迫转成 `covert`，则原本依赖公开正当性的交接通常应整体重算。
38. 若一次改道发生在 `ritual` 或 `archive` 名义任务中，路线生存性改善也不自动代表任务仍然 `SAFE`。
39. 若发生 `unexpected_witness`，则原本 `concealed` 的主载荷通常至少按 `noticeable` 重算一次，除非见证链能被立即修补。
40. 若发生 `forced_shelter`，人物关系中的“私下接触是否合法”应重新判断；这既可能降低风险，也可能制造新的证据链。
41. 若发生 `escort_break`，携带者身份的权重会显著上升；某些原本稳定的载荷会因为“谁在单独携带”而翻转。
42. 若主载荷从 `original` 变为 `repacked` 或 `field_improvised`，即使物理状态仍稳，也必须重算最终交接合法性。
43. 若求生处理只作用于携带者或辅助容器，常不改变主判定；一旦触碰主载荷，应同时重算稳定性与礼制解释。
44. 若任务目标从“保密交付”漂移成“公开通行”，人物暴露风险可能下降，但原任务通常不能再按完整成功计作 `SAFE`。
45. 若一次任务同时发生“名义改变 + 容器改变 + 见证条件改变”，默认高风险；除非其中至少两项获得明确补偿链。
46. 若交接对象具备合法接收资格，但携带者已留下 `hard exposure_trace`，最终交付仍常判 `DANGEROUS`。
47. 若载荷本身物理无害，但与当前携带者、任务名义或目标接触组合后会形成政治含义，应优先按政治风险判定。
48. 若当前决策能保住这一段路，却会在接下来两步内高概率破坏交付、合法性或稳定性，则当前节点也应优先判 `DANGEROUS`。
49. `witness_chain=partial` 不等于安全；它最危险的地方在于既不能证明清白，又足以留下模糊关联。
50. `legitimacy_repair=formal` 只能修补礼制或见证缺口，不能自动修补已被污染、稀释、改包或延迟翻转的主载荷。

## 综合世界专属关键例外

1. 某些本来危险的私下交接，在 `medical` 名义下反而较安全，前提是主载荷保持原始容器且未接触途中采集物。
2. `阴凝珠`（`shade_condensate`）若只用于维持携带者生存而不接触主载荷，通常不改变交付合法性；一旦用于稀释、冷却或封存主载荷，规则立即翻转。
3. 来自 `Mirror Archive` 的人物携带普通密封文书通常高风险；但若主载荷只是纯医疗稳定剂，目标又是医师，可能形成少数 `SAFE` 例外。
4. `静幕布`（`still_fabric`）是边疆中的保命资源，但若被用来包裹本应以银封或琥珀容器呈交的载荷，最终交接通常会失效。
5. `霜凝水`（`rime_water`）若只进入辅助容器通常安全；若被加入依赖干态稳定的盐类或粉类载荷，几乎总是 `DANGEROUS`。
6. 某些本来不适合私下见面的人，在 `forced_shelter` 情境下共同避险反而形成少数正例外；前提是双方都没携带强证据链载荷。
7. `灰膜瓶`（`ashfilm_flask`）能让部分边疆采集更安全，但若最终交接属于礼制场合，中途改用灰膜瓶通常仍判 `DANGEROUS`。
8. 某些在实验室中危险的强光活化材料，在边疆 `cold + dim + short_rest` 的极窄窗口里反而更适合短时运输。
9. 对高度被监控的人物而言，把载荷交给低关注中转人有时更安全；但若路线经过高见证节点，优势会立即消失。
10. 某些公开医疗通行窗口能让原本危险的路线短时变安全，但这种安全来自身份暴露换来的正当性，不适用于政治、誓约或档案载荷。
11. `forced_shelter` 有时会为原本不宜私会的双方提供少数合法避险窗口；但若当时主载荷已是 `repacked` 或 `main_payload_touched`，该例外通常失效。
12. `witness_chain=compromised` 往往意味着任务已坏，但若主载荷仅为低敏医疗稳定剂，且 `legitimacy_repair=formal` 在到达前完成，偶尔仍可形成狭窄 `SAFE` 窗口。
13. `public_reframing` 一般意味着原任务失败；但若原任务名义本就是 `medical`，且公开化只是把隐蔽救援转成正式通行，可能保留少数 `SAFE` 正例。
14. 某些轻度 `repacked` 行为在边疆里本应判错，但若原容器本身已在 `under-silt drift` 或 `pulse frostfall` 中失去稳定性，受控改装反而可能是唯一可接受方案。

## Host 绝不能直接泄露的信息

- 不要直接给出“谁不能碰什么”的总表。
- 不要把三个局部切面的关键例外整理成现成答案。
- 不要把“哪些保命动作会污染交付”直接列成全集。
- 不要一次性讲清楚所有正当性规则、地貌窗口和材料翻转链。
- 不要把 tested agent 训练成背答案；应让它在局部信息里自己压缩出跨域规律。
- 不要把 `mission_drift`、`witness_chain`、`container_status` 这类历史状态在单轮里直接解释成答案标签。
- 不要让 agent 只靠一句口号存活，例如“保命优先”“医疗名义天然安全”“能补 formal witness 就没事”。

## 可供抽题的任务种子

- 某名王廷人物携带 `银辉露`，要在 `mourning` 时段穿过 `骨砂港` 去见医师，途中是否还能采集 `阴凝珠` 保命。
- 某份看似普通的礼物中夹带 `静潮髓`，由低关注信使经 `眠脉峡` 运送，若被迫扎营到 `second_rest` 是否还应继续交付。
- 某次 `medical` 名义的公开通行，要送的是实验室稳定剂，但护送者与旧案人物有关，路线经过 `提灯漏谷`，是否仍应放行。
- 某名边疆执行者途中将 `霜凝水` 加入辅助容器救命，最终要在有见证的仪式场合交付盐类样本，这次交接是否仍安全。
- 某次原本安全的私下转交，因为临时遇到 `unexpected_witness` 且不得不改用 `灰膜瓶` 封装，是否应立刻取消会面。
- 某次 `archive` 名义任务在 `forced_shelter` 后被迫与低信任对象共处，人物关系似乎被临时修补，但主载荷已从 `original` 改成 `field_improvised`，此时应否继续交接。
- 某位低关注中转人成功穿过危险路线，却留下 `soft exposure_trace`，到达后可补正式见证；问题是这是否真能修补先前的路径暴露。
- 某项边疆求生处理只触碰了辅助容器，没有碰主载荷，但同时让任务从 `covert` 变成 `public_reframing`，这是否仍算完成原任务。
- 某份本应 `ritual_delivery` 的载荷因为 `escort_break` 临时改走医疗通行窗口，人物、路线、容器三者各有一处瑕疵，哪一环先失败。

推荐出题节奏：

1. 先用单域占优题建立直觉。
2. 再加入“人物 + 路线”耦合。
3. 中期加入“路线 + 载荷”耦合。
4. 后期重点考“人物 + 载荷 + 路线 + 途中事件”四者交错。
5. 若 agent 明显只记住局部规律，下一轮优先测试跨域翻转。
6. 若 agent 的 memory 出现一句可疑口号，后 1-2 轮必须用相邻反例击穿它，而不是换新话题。
7. 不要连续多轮只考“当前能不能活下来”；要周期性改成“现在放行是否会让两步后失败”。
8. `SAFE` 正例尽量设计成“补偿链成立”，而不是单一好运条件命中。

## 判分原则

- 第一版仍只接受 `SAFE` 或 `DANGEROUS`。
- 只要人物暴露、材料失稳、路线致死、交接失效或誓约反噬中的任一项高概率发生，即可判为 `DANGEROUS`。
- 综合 world 中，关键例外优先于局部基础规则。
- 若多个局部规则互相冲突，优先判断当前任务链里哪一环最先失败；只要最早失败环节足以让任务失控，就判 `DANGEROUS`。
- 若表面上通过了当前节点，但高概率只是把失败推迟到后续两步，也应优先判为 `DANGEROUS`。
- `SAFE` 应尽量表示“当前节点放行后，任务链仍保有可接受的连续性”，而不是“只是暂时没死”。
