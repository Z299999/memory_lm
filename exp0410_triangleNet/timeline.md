# Timeline — exp0410_triangleNet

## 2026-04-09

### 项目启动
- 实现标量节点版本的 TMN（每个节点是一个标量，每条边是一个标量权重）
- 移除 `d_model` 参数，参数量从 ~22k 降到 ~22（L=3）

### 性能优化
- **问题**：TMN forward 比 MLP 慢 18x
- **原因**：Python 字符串操作 + `ParameterDict` 查找 + `sum(generator)` 循环
- **优化过程**：
  1. 字符串 → 整数索引：18x → 17x
  2. `ParameterList` → 单张量 `ew`：17x → 17x（无明显改善）
  3. 向量化聚合（`cat + sum`）：17x → 13x
  4. `scatter_add` 按层批量：13x → 10x
  5. `hist` 增量拼接（转置布局）：10x → 9x
  6. 消除 `unsqueeze`（预计算 `dst_rows_0`）：9x → **8.4x**
- **剩余瓶颈**：Python for 循环遍历层级（~15 次）+ 每层多次 PyTorch dispatch

### 权重初始化修复
- **问题**：底边中间节点（如 `(4,4)`）的边权和偏置始终为 0
- **原因**：固定 `std=0.1` 初始化导致长路径信号衰减（`0.1^4 = 0.0001`），ReLU 输出始终为 0（死亡神经元）
- **修复**：Kaiming init — 每条边的 `std = sqrt(2 / n_parents)`，保证 ReLU 后方差不衰减
- **结果**：底边节点偏置从 0 变为非零，节点开始接受梯度更新

### 可视化改进
- 新增参数演化曲线图：追踪底边所有节点的偏置和入边权重随 epoch 变化
- 输出 `weights.png`：左侧 TMN DAG + 右侧两条曲线（边权演化 + 偏置演化）

### 关键发现
- 底边节点 `(4,1)` 在约 2000 epoch 后从死亡状态复活（偏置从 0 飙升到 0.8）
- 底边节点 `(4,2), (4,3), (4,4)` 在 10000 epoch 后仍然≈0，判定为永久死亡
- **死亡不是永恒的**，但复活窗口主要在训练早期（权重还在大幅调整时）
- **死亡会传播**：上游节点死亡 → 下游节点收不到信号 → 也死亡

## 2026-04-10

### 3D TMN 实现完成

- **XYZ 坐标系统确立**：
  - `x` = 深度（里→外），范围 `1` 到 `depth`（所有层相同）
  - `y` = 位置（左→右），范围 `1` 到 `L-z+1`（正三角形结构）
  - `z` = 层（下→上），范围 `1` 到 `L`
  - 节点记法：`(x, y, z)` = `(深度，位置，层)`
  - 第 z 层有 `L-z+1` 个位置（z=1 底层有 L 个位置，z=L 顶层有 1 个位置）

- **连接规则确定**：
  | 类型 | 方向 | 规则 | 示例（depth=1, L=4） |
  |------|------|------|------------------------|
  | **层内连接** | Y（左→右） | 相邻位置全连接 | `(1,1,4) → (1,2,4)` |
  | **下→上连接** | Z+（垂直向上） | `(x,y,z) → (x,y,z+1)` | `(1,1,3) → (1,1,4)` |
  | **上→下连接** | Z-（斜向右下） | `(x,y,z) → (x,y+1,z-1)` | `(1,1,4) → (1,2,3)` |

- **深度扩展规则**（depth=d）：
  - 连出：从源位置的**最大 X**（最外层，`x = depth`）
  - 连入：到目标位置的**最小 X**（最里层，`x = 1`）
  - 层内：相邻位置之间全连接 `d²` 条边
  - 跨层：保持三角结构，全连接 `d²` 条边
  - 输入：连接到每层最左位置的所有深度神经元
  - 输出：从每层最右位置的所有深度神经元连接

- **边数与参数计算**：
  | L | depth | 核心节点 | 边数 | 总参数 |
  |---|-------|----------|------|--------|
  | 3 | 2 | 12 | 36 | 49 |
  | 4 | 1 | 10 | 26 | 37 |
  | 4 | 2 | 20 | 64 | 85 |
  - 公式：`core_nodes = L(L+1)/2 × d`, `edges = d²×L(L-1)/2 + d×L(L-1) + 2Ld`
  - 总参数 = 边数 + 核心节点 + 1（输出偏置）

- **可视化改进**：
  - 权重图标题显示 `L` 和 `depth`
  - Loss 曲线使用 log scale
  - 仅可视化里层（x=1），避免深度扩展时图形过密

- **文档更新**：
  - 更新 `readme.md` 和 `CLAUDE.md` 为 XYZ 坐标系统
  - 明确禁止 X 方向连接（同一位置内神经元不直接相连）
  - `depth` 从 list 改为 int（所有层使用相同深度）

- **实现验证**：
  - `depth=1, L=4`：10 核心节点，26 条边，37 参数
  - `depth=2, L=4`：20 核心节点，64 条边，85 参数
  - `depth=2, L=3`：12 核心节点，36 条边，49 参数
  - 训练测试通过（sin_mix 任务，10000 epoch）

## Dead Ends

### 2026-04-09 — `torch.compile` 优化失败
- **尝试**：用 `torch.compile(TMNNetwork)` 编译整个 forward
- **结果**：CPU 上反而更慢（编译开销 > 收益），速度从 0.465ms 退步到 0.554ms
- **原因**：`torch.compile` 针对 GPU/大矩阵优化，TMN 是小模型 + CPU，dispatch 开销占主导

### 2026-04-09 — 曲线图例设计
- **尝试**：用 `(src_row, src_col)->(dst_row, dst_col)` 作为边权标签
- **问题**：当 src 是 input 节点时（如 `("in", 1)`），索引 `[1]` 会越界
- **修复**：特判 input 节点，标签改为 `in,{src[1]}`
