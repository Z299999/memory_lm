# exp0422 — DQN CartPole

## 实验目标

在自定义 CartPole 环境上实现最简 DQN，理解强化学习基础概念：Q 函数、TD 误差、replay buffer、target network、epsilon-greedy 探索。

## 算法

**DQN（Deep Q-Network）**，两张结构相同的网络：

| 网络 | 结构 | 作用 |
|------|------|------|
| Q-network | 4 → 64 → 64 → 21 | 选动作（argmax Q），被训练 |
| Target network | 4 → 64 → 64 → 21 | 计算 Bellman 目标值，定期同步 |

- 探索：epsilon-greedy，从 ε=1.0 指数衰减到 0.01
- 经验回放：replay buffer 容量 10000，batch size 32
- 损失：MSE( Q(s,a), r + γ·max Q_target(s',a') )

## 动作空间

21 个离散整数动作，`action ∈ {0, 1, ..., 20}`，映射为力 `F = action - 10 ∈ {-10,...,+10} N`。

## 奖励函数

在 CartPole 原始奖励（每步 +1）基础上加 6 项惩罚：

```
r = 1 - w₁·x² - w₂·ẋ² - w₃·θ² - w₄·θ̇² - w₅·ẍ² - w₆·θ̈²
```

物理引擎由 `cartpole_custom.py` 从头实现（Lagrangian 动力学，Euler 积分）。

## 网络容量实验

| 网络 | 参数量 | 后100集均值 | 结论 |
|------|--------|------------|------|
| [16, 16] | 709 | 152 ❌ | 学会后崩，容量不足 |
| [32, 32] | 1909 | 484 ✅ | **最小可 solved 网络** |
| [64, 64] | 5845 | 485 ✅ | 最稳定，test 497.68 ±0.29 |

**边界在 [16,16] 和 [32,32] 之间**：网络太小无法稳定记住策略。

## 局限性

1. **Catastrophic forgetting**：epsilon 衰减到底后，agent 在固定策略上反复更新，容易把好的权重覆盖掉。reward 曲线在学会后仍会崩塌。

2. **Replay buffer 污染**：早期随机探索的低质量经验和后期好经验混在同一个 buffer 里，可能干扰训练。优先经验回放（PER）可以缓解。

3. **离散动作只能近似物理**：力只能选整数值，精细控制受限。连续动作空间对物理控制更自然（→ exp0430 连续 PPO）。

4. **Target network 更新是脆弱超参**：更新太频繁导致训练不稳定，太慢导致 Q 值过时。默认每 100 集更新一次，未经仔细调参。

5. **单 seed，无统计验证**：所有结果基于单次训练。多 seed 对比（exp 框架支持 num_seeds 参数）才能得出可靠结论。
