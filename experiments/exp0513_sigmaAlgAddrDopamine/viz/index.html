exp0513 Dashboard Help

Workflow

  • Edit the configuration fields in the left panel.
  • Press Confirm to validate the config and refresh the dopamine/graph preview.
  • Press Run only after a successful confirm. If you edit the form again, confirm once more before running.
  • Use Stop to request an epoch-boundary stop, Refresh to pull the latest dashboard state, and Reset to restore the default configuration form.

Legend

  • Blue: input node
  • Gray: hidden neuron
  • Green: dopamine hidden node
  • Red: output node
  • Blue-highlighted edge: controlled by the selected dopamine node
  • Orange edge: currently selected edge

Interaction

  • Click a dopamine hidden node to highlight every forward edge it controls.
  • Click any node or edge to see a small value popup at the click location.
  • The right panel keeps the complete detail view while the graph popup shows only the key values.

Diagnostics

  • Preview-only states show initialized edge weights immediately.
  • Node activations appear after the first validation pass of a real run.
  • Local loss tracks only the current continuation run; global loss spans all resumes.

Resume semantics

  • Resume latest model.pt automatically points at the newest checkpoint in runs/.
  • resume_from stays editable only when Resume latest model.pt is unchecked.
  • epochs means how many additional epochs to continue, not the total historical epoch count.

Dopamine coverage semantics

  • c is the average edge coverage: how many dopamine nodes cover an edge on average.
  • m is the dopamine node count. If left blank, the dashboard recommends a value from the current architecture.
