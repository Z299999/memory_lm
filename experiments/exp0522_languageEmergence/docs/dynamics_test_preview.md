# 语言系统的动力学性能分析（测试预告）

> 本文是**预告 / 计划**，不是结果报告。目的：把"语言通道 = 受常数脉冲驱动的离散动力系统"这一视角固化下来，给出统一的性能标尺、可证伪的假设，以及待跑的测试清单。结果出来后另起报告文件。

## 0. 一句话

把固定语言通道当成一个**受常数脉冲驱动的离散动力系统**，用"线性化特征值离单位圆多远"这一把尺子，同时衡量它的**性能**（能不能持续追踪）与**脆弱性**（会不会发散成 NaN）——二者是同一枚硬币。

## 1. 系统形式化

唯一外部输入是常数脉冲 $x_t=1$。递推状态是 message $m_t\in\mathbb{R}^{17}$（外加标量 error $e_t$）：

$$
\ell_t = m_{t-1}(I+D),\qquad
h_t = \mathrm{MLP}_{\tanh,\,res}\big([\,1,\;e_{t-1},\;\ell_t\,]\big),\qquad
y_t = \mathrm{decode}(h_t)
$$
$$
m_t = R^\top h_t^{\text{(concat)}}\quad(\text{固定稀疏带符号读出, 每列单位范数}),\qquad
e_t = y^*_t - y_t\ \ (\text{detach, 可缩放})
$$

两种读法：

- **自治振荡器**（`sole_speech`，掐掉 error）：$m_t = G(m_{t-1})$，一个 17 维自治离散映射，必须靠自身动力学吐出准周期信号。
- **闭环控制器**（`full`）：$m$ 是内部振荡器状态，$e$ 是把它锁回真实相位的反馈，结构上类似 PLL / 观测器校正。

可学习的只有 MLP 主体和 carry 矩阵 $D$；读出 $R$ 固定（见 [PLAN.md](PLAN.md) 的 fixed sparse signed readout）。

## 2. 任务对系统的要求

目标 $y^*_t=\sum_k a_k\sin(\mathrm{freq}_k\cdot\omega t)$，$\omega=2\pi/32$，分量周期 $P_k=32/\mathrm{freq}_k$ 步：

| freq | 周期(步) | 振幅 | 模态角 $\theta_k=2\pi/P_k$ |
|---|---|---|---|
| 1.0 | 32 | 1.0 | 0.196 |
| 0.7 | ~46 | 2 | 0.137 |
| 0.05 | 640 | **3** | 0.0098 |
| 0.029 | ~1100 | 1.5 | 0.0057 |
| 0.01 | 3200 | 1 | 0.0020 |
| 0.004 | 8000 | 2.1 | 7.9e-4 |
| 0.0013 | ~24600 | 1 | 2.6e-4 |
| 0.0005 | **64000** | 2 | 9.8e-5 |

两个硬约束：

- **刚性 ~2000×**：周期从 32 到 64000 步。
- **容量**：8 个分量 ≈ 16 个振荡模态，message 只有 17 维——几乎卡在下限。

## 3. 中心命题：性能 = 稳定裕度

要持续输出某频率的振荡，系统线性化的特征值必须**贴在单位圆上**对应角度 $\theta_k$。于是三种 regime：

- $|\lambda|<1$：振荡衰减 → 漂回均值（欠激励，跟不住）。
- $|\lambda|\approx 1$：理想振荡器，但**临界稳定 = 脆弱**。
- $|\lambda|>1$：指数增长 → 爆炸 → **NaN**。

由此推出两条可检验的论断：

1. **能力上限来自刚性。** 周期 64000 的慢模态要求 $|\lambda|$ 落在距 1 约 $10^{-5}$ 的窄缝里，几乎不可能纯靠内部动力学维持 → 慢分量（尤其 freq 0.0005、振幅 2）大概率是**靠 error 反馈每步重新注入**，不是内生的。快分量（周期 32–46）才可能内生。

2. **"训练太好就 NaN"的动力学解释。** 自治映射被推到单位圆边缘以维持振荡；模型越准 $\Rightarrow e_t\to 0 \Rightarrow$ **反馈阻尼消失** $\Rightarrow$ 只剩裸自治映射，恰好被调在临界点上，长窗 BPTT 一推就越过 1 → 失稳。即：**error 通道平时兼任阻尼器，恰在性能登顶（误差归零）时关掉，系统裸奔到不稳定一侧。** 这解释了实测里 `steps` 一跳满 2000（窗口拉满、误差极小）就爆 NaN 的现象。

## 4. 测试清单（预告）

每项给出：目的 / 方法 / 预期假设 / 判读。所有测试都在**同一个没爆的 checkpoint** 上做。

### A. carry 矩阵 $(I+D)$ 的特征谱
- **方法**：复平面画 $(I+D)$ 的 17 个特征值；标注 $|\lambda|$ 与辐角。
- **预期**：出现成对复特征值贴近单位圆，辐角对上表中的 $\theta_k$（至少快分量）；谱半径 $\ge 1$ 即潜在失稳源。
- **判读**：$(I+D)$ 只是线性 carry，是廉价**一阶代理**；严格版见 B。

### B. 沿轨迹的 Jacobian / Lyapunov 谱
- **方法**：沿一段 rollout 累乘单步 Jacobian $J_t=\partial m_t/\partial m_{t-1}$，估领头 Lyapunov 指数 $\lambda_1$ 和谱半径包络。
- **预期**：健康追踪时 $\lambda_1\approx 0^-$（临界略偏稳定）。
- **判读**：$\lambda_1>0$ = 要爆（NaN 预警）；$\ll 0$ = 过阻尼、跟不住。这是"既准又脆"的定量刻度，也是离 NaN 多远的早期信号。

### C. 自由跑（`sole_speech`）长程 FFT
- **方法**：掐掉 error，自由跑 $\gg 64000$ 步；对输出做 FFT，并看各目标频带的振幅包络。
- **预期**：快分量频率峰值保留、慢分量衰减或漂移；包络斜率 = 各模态有效阻尼。
- **判读**：直接验证命题 1（慢分量靠反馈）。

### D. full vs sole_speech vs dim 的反馈依赖
- **方法**：同一 checkpoint 跑 `full`、`sole_speech`、`dim(...)`，按频带比较追踪误差。
- **预期**：慢分量误差对"掐反馈"极敏感，快分量不敏感。
- **判读**：量化"准确度里多少是反馈喂的、多少是内生的"——即闭环控制性能。

### E. 扰动恢复（`blink` / `stutter`）
- **方法**：用现有 eval 条件制造瞬时中断，测恢复时间常数。
- **预期**：若极限环是**吸引子**则会回弹；若是**临界中心**则扰动累积、不回弹。
- **判读**：error 反馈正是把"中心"变成"吸引子"的耗散项；对比 full 与 sole_speech 的恢复差异。

### F. 容量裕度（`language_dim` 扫描）
- **方法**：扫 $k\in\{8,12,16,17,24,32\}$，看追踪误差与稳定性。
- **预期**：$k<16$ 明显追不动（模态不够）；$k\gtrsim 16$ 收益递减。
- **判读**：验证"17 维卡在下限"的容量论断。

## 5. 健康 vs 脆弱的判据

一个**又好又稳**的系统应满足：

- A/B：特征值贴近但**不越过**单位圆（$\lambda_1\lesssim 0$）。
- C：内生维持快分量，慢分量明确标注为"反馈承担"。
- E：blink/dim 后能回弹（吸引子），而非扰动累积（中心）。

典型**脆弱**画像：性能极好（val 极低）+ $\lambda_1$ 紧贴 0 甚至 $>0$ + 误差一掉到 0 窗口就拉满 → 随时 NaN。这正是当前主线 run 的状态。

## 6. 依赖与下一步

- **前置**：需要一个跑稳、没 NaN 的 checkpoint（或当前 run 爆掉前的 epoch milestone，如 epoch-100）。
- **稳定化手段**（与本分析的机制对应，详见 [README](../README.md) 的 event_triggered / tail_dim 说明）：
  - 砍 BPTT 深度（`event_triggered` 的 max 调小）——直接压低 B 里的 $\lambda_1$ 暴露。
  - 非有限 step 守卫（loss 非有限就跳过 `optimizer.step()`），避免单个坏窗口把权重永久打成 NaN。
- **产出**：A、C 先行（最便宜、信息量最大），随后 B/D/E，最后视需要做 F。
