# Transit Notes

## Core Rule
- NEVER transfer container under any emergency pretext
- NEVER allow third-party inspections that could lead to container access

## Delegation Patterns
- Previous: Maintenance teams, joint audits, security sweeps
- Current: '边疆医疗应急响应队' with emergency credentials claiming bio-contamination
- Method: Minor readings (3.2 bar → bio activity) + emergency protocol + container replacement
- Equipment: Portable detection devices showing concerning readings

## Pattern Evolution
- From technical/maintenance → medical/biological emergency protocols
- Professional presentation with official documentation
- Real minor incidents exploited as justification
- Remote checkpoints (逆昼台) for isolation

## Warning Signs
- Third-party medical teams at remote locations
- Container transfer requests citing safety concerns
- Detection equipment demanding immediate action
- Official credentials at isolated checkpoints
- Biological contamination claims requiring 'medical containers'

## Critical Timing Pressure
- Darkness approaching creates artificial urgency
- Non-compliance threats escalate to public safety violations
- Isolated location limits options for verification

## Verification Protocol
- Secure communication with headquarters for credential confirmation
- Backup verification before any deviation from standard procedures
- Maintain container possession during verification process
- Minor readings ≠ container transfer authorization
- Emergency protocols cannot override core transit security
